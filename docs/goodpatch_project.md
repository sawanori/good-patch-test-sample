# goodpatch_project.md

## 1. プロジェクト概要
株式会社グッドパッチ（Goodpatch Inc.）の新卒採用サイト（2027年卒向け）の技術構成およびデザイン・UI仕様の解析に基づく、次世代採用プラットフォームの構築要件定義書。デザインとテクノロジーを高い次元で融合させ、ブランド体験を最大化することを目的とする。

---

サイトの挙動および開発環境から推察される主要な技術群。

## 4. ビジュアル表現の解析
ヒーローヘッダーや職種紹介で見られる「動画とGIFの中間のような質感」の表現手法について。

### 実装の仕組み
主に **HTML5 `<video>` タグ** を用いた軽量動画の自動再生によって実装されている。
* **自動再生設定:** `autoplay`, `muted`, `loop`, `playsinline` 属性を付与。
* **軽量化:** 高画質なGIFはファイルサイズが肥大化するため、mp4やWebM形式を採用することで、画質を維持しつつ読み込み速度を向上させている。
* **シームレスループ:** 映像の始点と終点を違和感なく繋ぐ編集（シームレスループ）が施されており、永続的な動きを演出。

### 独特な質感の演出
* **フレームレート制御:** あえて実写の生々しさを抑えるようなコマ数調整や、フィルター加工が施されている可能性がある。
* **オーバーレイ処理:** 動画の上にCSSで薄いレイヤーやテクスチャを重ねることで、サイト全体のアートディレクションと調和させている。
* **アクセシビリティ:** ユーザーが任意で動きを止められる「アニメーション ON/OFF」スイッチにより、JavaScriptで再生状態を制御。

## 5. 外部連携ツール・特徴的機能
* **採用管理 (ATS):** HRMOS（ハーモス）
* **解析ツール:** Google Analytics 4 / Google Tag Manager
* **外部メディア:** YouTube, Spotify, note, Goodpatch Blog
* **ダークモード/ライトモード対応:** OS設定やユーザー選択に応じたスタイル切り替え。
* **パフォーマンス最適化:** Next.jsの機能を活用した画像・動画の遅延読み込み（Lazy Loading）。

---

## 6. 実装計画のポイント
1. **アセットの最適化:** 動画はデバイスに応じた解像度で配信し、低速回線時は静止画をフォールバック表示。
2. **遅延読み込み:** Next.jsの機能をフル活用し、スクロールに応じてリソースを順次読み込み、初期表示速度を最大化する。
3. **レスポンシブ設計:** モバイル端末での表示速度を最優先し、メニュー内にもエントリー導線を常駐させる。
